COMPONENT SUMMARY & RESPONSIBILITIES: -------------------------------------------------------------------------------- * CONFIGURATION (`config.js`): Acts as the single source of truth for shift settings, counts, and compliance thresholds. Modifying shift limits or weekend structures here safely updates the entire application behavior dynamically without changing the engine code. * SCHEDULER ENGINE (`schedulerEngine.js` & `shiftAllocator.js`): Executes a multi-pass optimization loop. It reserves static leave days first, structures the mandatory bi-weekly Friday/Saturday weekend blocks, guarantees the post-night shift rest intervals, and distributes remaining 'OFF' slots. Finally, standard daily shifts (Morning, Evening, Night) are allocated. * FAIRNESS & SCORING ENGINE (`fairnessEngine.js` & `scoring.js`): Keeps a dynamic balance matrix tracking cumulative shift loads for every staff member. The allocator checks these live scores before assigning shifts to prevent consecutive weekends or consecutive heavy shift distributions. * EXCEL EXPORT ENGINE (`excelExport.js`): Utilizes SheetJS (XLSX) to convert the multi-dimensional schedule matrix into a highly polished native spreadsheet. It automatically injects conditional styles (e.g., highlighting 'OFF' and 'NIGHT' blocks) and runs custom cell width auto-fitting formulas. * USER INTERFACE (`dashboard.js` & `tableRenderer.js`): Generates a clean, modular grid directly inside the browser DOM. Includes interactive submission dialogs for personnel profiles and customized leaves or day-off requests, flashing instantly upon successful calendar updates. 2. CORE SYSTEM WORKFLOW: -------------------------------------------------------------------------------- [1] Initialization: Loads pre-configured employee rosters and custom options from 'dataStore.js'. [2] Hard Constraints Pre-Mapping: Maps approved annual leaves and individual date requests into the grid, locking those cells securely from future modifications. [3] Weekend Off Rotation: Divides the month into alternating 2-week windows, assigning every team member a locked consecutive Friday-Saturday off block to maximize fatigue recovery. [4] Night-Shift Safeguards: Whenever a Night shift is scheduled, the engine automatically inserts the mandatory 'OFF' day right after it, preventing erratic work cycles. [5] Uniform Off-Day Distribution: Spreads the remaining 'OFF' balance smoothly throughout the month so that no worker gets clustered rest periods at the start or end of the month. [6] Dynamic Balancing: Fills in the empty slots with Morning and Evening shifts using live metric scores to guarantee standard workloads across all staff ranks. 3. SETUP & IMPLEMENTATION INSTRUCTIONS: -------------------------------------------------------------------------------- To deploy this system instantly in your local workspace or server environment: Step 1: Save the code into a unified file (e.g., `index.html`) or divide it into the target directory tree defined in your structural requirements. Step 2: Ensure your computer or client browser has an active internet connection to pull the SheetJS library from the secure CDN link embedded in the header. Step 3: Open `index.html` directly using any modern web browser, or launch it through a local development web server (such as VS Code Live Server). Step 4: Use the interactive control dashboard to seamlessly add new team members, input custom leave dates, run the engine, and export flawless Excel sheets. 4. ERROR PREVENTION & TROUBLESHOOTING: -------------------------------------------------------------------------------- * Excel Download Blocked: If the file download does not launch when clicking the export button, verify that your browser is not blocking multiple file downloads or popups from unrecognized local sources. * Empty Cells or Shift Gaps: This can happen if the staff count is too low relative to your daily minimum coverage constraints. Ensure a healthy staff-to-shift ratio (ideally 5+ active employees for standard 3-shift coverage patterns) to provide the allocation engine with enough variables to optimize perfectly. ================================================================================
